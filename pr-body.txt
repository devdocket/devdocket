## Summary

Simplifies the Focus view state model by replacing the `Blocked` and `WaitingOn` states with a single `Paused` state. This reduces complexity in the state machine, UI commands, and context menus without losing expressiveness.

Closes #34

## Changes

### State model (6 states, down from 7)
- **Removed:** `Blocked`, `WaitingOn`
- **Added:** `Paused` — covers both use cases

### Commands
- `workcenter.blockItem` / `workcenter.markWaitingOn` → `workcenter.pauseItem`
- `workcenter.unblockItem` → `workcenter.resumeItem`

### UI
- Focus view context values: `blocked` / `blocked.hasUrl` → `paused` / `paused.hasUrl`
- Icon: `debug-pause` (⏸) for paused items
- Description: "⏸ paused" instead of "⛔ blocked" / "⏳ waiting"

### Data migration
- `jsonTaskStore.ts` automatically migrates persisted `Blocked`/`WaitingOn` states to `Paused` on load

### Tests
- Updated all existing tests referencing old states
- Added 3 new migration tests (Blocked→Paused, WaitingOn→Paused, persistence)

### Documentation
- Updated `docs/extension-api.md`, `AGENTS.md`, and `.squad` agent docs to reflect 6-state model
